# bouncycastle
---------------------------------------------------------------
## 대칭키(대표적인 AES & DES)
```
-대칭키 암호시스템은 송신측과 수신측간에 각각 <u>암호화와 복호화를 위하여 동일안 비밀키를 사용</u>한다.
-이 키는 비밀 통신망 또는 직접전달 등의 안전한 전송방식을 통해 사전에 전달되어야 한다.
-대칭키 암호화의 방식은 암복호화 <u>속도가 빠른 이점이 있으나 키관리에 어려움</u>이 있다.
```
## 대칭키 특징
```
-암호화 연산 속도가 빨라서 효율적인 암호 시스템을 구축할 수 있음.
-알고리즘을 쉽게 개발할 수 있고 컴퓨터 시스템에서 빠르게 동작함.
-송, 수신자 간에 동일한 키를 공유해야 하므로 많은 사람들과의 정보 교환 시 많은 키를 생성, 유지, 관리해야 하는 어려움이 있음.
```
----------------------------------------------------------------
## 비대칭키(공개키 암호시스템 RSA)
```
-대칭키 암호시스템에 반해 비대칭키(공개키 암호시스템)는 <u>암호화와 복호화를 위해 서로 다른 키를 사용</u>한다.
-다른 키를 공개키라고 하며 공개된 장소에 쉽게 접근이 가능하도록 보관한다.
-암호화 할 때 사용하는 공개키는 공개된 시스템에 전화번호처럼 공개하고 복호화 할 때 사용하는 비밀키는 안전하게 보관하여야 한다.
-비대칭키는 암/복호화 할때의 Key값이 다르다. 대표적으로 RSA라는 암호화가 있으며, 암호화 할때의 키를 공개키 복호화 할때의 키를 개인키라 한다.
```
## 비대칭키 특징
```
-키 동기화 문제가 적음.
-느리고 HTML에 약함.
-공개된 장소에 쉽게 접근이 가능하도록 보관함.
-암호화 할 때 사용하는 공개키(public key)는 공개된 시스템에 전화번호처럼 공개하고 복호화할 때 사용하는 비밀키(private key)는 안전하게 보관되어야 함.
```
------------------------------------------------------------------
## 요약
```
-암/복호화 할때의 Key의 값이 같으면 **대칭키** 다르면 **비대칭키**라고 한다.
-대칭키를 사용하는데 둘중 하나라도 유출 되면 암호문은 제역활을 할 수 없게 된다.(비밀리에 키를 공유해야 하므로 비밀키)
-비대칭키를 사용하는데 하나는 공개되어도 상관없는 키(공개키), 다른 하나는 공개되면 안되는 키(비밀키) 이렇게 2개의 키를 가지게 된다. 주로 공개키는 평문을 암호화하는데 사용되고 비밀키는 되돌리는데 사용된다.
```
